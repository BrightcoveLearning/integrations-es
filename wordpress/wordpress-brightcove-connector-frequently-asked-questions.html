Conector de WordPress-Brightcove: Preguntas frecuentes

Este tema contiene preguntas frecuentes sobre el conector de Wordpress-Brightcove.

¿Puedo ejecutar el complemento en una instalación de WordPress que no es de acceso público?

Sí, funcionará tanto si es público como si no, sin embargo, las funciones que requieren una devolución de llamada a su sitio, como actualizaciones de estado, carga de videos, etc., no estarán disponibles.

 ¿Hay filtros para los desarrolladores de complementos / temas? 

brightcove_account_actions = [editar, eliminar]; // Qué acciones están disponibles al manipular una fuente de Brightcove.

¿Funcionará esto en varios sitios?

Sí lo será.

¿Puedo usar más de una cuenta de Brightcove?

Sí, puede agregar fuentes de muchas cuentas de Brightcove si lo desea.

¿Puedo establecer diferentes permisos para los usuarios?

Si. Puede encontrar una matriz de permisos de usuario en este documento. Para cambiar los roles de los usuarios, recomendamos usar la función add_cap de WordPress: https://codex.wordpress.org/add_cap

¿Cómo funciona la sincronización?

El complemento simplemente extrae información directamente de la API de Brightcove para mostrarla y no sincroniza videos localmente.

¿Cómo puedo aumentar el tamaño máximo de archivo de carga?

El tamaño máximo del archivo está determinado por su servidor web y la configuración de PHP. Debe establecer el valor de upload_max_filesize y post_max_size en su php.ini. php_ini_loaded_file () puede ayudarlo a encontrar dónde se encuentra su PHP.ini.

¿Cómo habilito el seguimiento de cambios para poder ver quién actualizó un video y cuándo lo hizo?

Cree un campo personalizado de tipo 'texto' con un nombre interno de '_change_history'. Siempre que se actualice un video, el nombre de usuario y la hora actual se agregarán a una lista de cambios registrados en este campo.
