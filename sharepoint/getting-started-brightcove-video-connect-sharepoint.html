    Instalación de Brightcove Video Connect para SharePoint

    Antes de comenzar la instalación, revise la siguiente información acerca de los requisitos previos y permisos:

    • Antes de que un usuario pueda agregar una aplicación para SharePoint, un miembro del grupo Administradores de granja debe configurar el entorno para admitir aplicaciones para SharePoint. Para obtener más información, vea Configurar un entorno para aplicaciones para SharePoint 2013 o SharePoint 2016 o SharePoint 2019.
    • Un usuario debe tener los permisos Administrar sitio Web y Crear subsitios para agregar una aplicación para SharePoint. De forma predeterminada, estos permisos sólo están disponibles para los usuarios que tengan el nivel de permiso Control total o que se encuentren en el grupo del propietario del sitio.
    • Cuando un usuario agrega una aplicación para SharePoint, la aplicación solicita permisos que necesita para funcionar (por ejemplo, acceso a Buscar o crear una lista). A los usuarios que no tienen esos permisos se les informa de que no tienen suficientes permisos y que la aplicación no se puede agregar. El usuario puede ponerse en contacto con un administrador del sitio o del conjunto de servidores para ver si el administrador puede agregar la aplicación.
    • Un usuario que inició sesión en un sitio como la cuenta del sistema no puede instalar una aplicación. La cuenta del sistema no puede importar licencias de aplicación porque esto podría dar lugar a problemas de rendimiento.

    Agregar la aplicación Brightcove Video Connect al catálogo de aplicaciones

    Configuración del catálogo de aplicaciones

    Para instalar el conector de Brightcove, el primer paso es configurar el catálogo de aplicaciones en SharePoint. Utilice las referencias a continuación para completar este paso.

    • Administrar el catálogo de aplicaciones en SharePoint 2013
    • Administrar el catálogo de aplicaciones en SharePoint 2016
    • Administrar el catálogo de aplicaciones en SharePoint 2019

    Agregar la aplicación Brightcove Video Connect al catalog

    Una vez configurado el catálogo de aplicaciones, siga los pasos que se indican a continuación:

    1. Compruebe que la cuenta de usuario que está realizando este procedimiento es miembro del grupo Propietarios de sitios o diseñadores del catálogo de aplicaciones.
    2. En el sitio de catálogo de aplicaciones, haga clic en la lista Aplicaciones para SharePoint .
    3. En la página Aplicaciones para SharePoint , haga clic en nuevo elemento.
    4. En el cuadro Elegir un archivo , haga clic en Examinar y, a continuación, busque la carpeta que contiene el archivo de BrightCoveConnectorApp .
    5. Seleccione la aplicación y, a continuación, haga clic en Abrir.
    6. Haga clic en Aceptar para cargar la aplicación.
    7. En el cuadro Detalles del elemento, proporcione el nombre, el título, la descripción corta, la URL del icono y otras opciones de configuración para la aplicación. Asegúrese de que la casilla Habilitado esté activada para que los usuarios puedan ver la aplicación en sus Sitios. Puede activar la casilla Destacados para mostrar la aplicación en la vista Contenido destacado del Catálogo de aplicaciones.
    8. Haga clic en Guardar.

    Instalación de la aplicación Brightcove Video Connect en un sitio

    Siga los pasos que se indican a continuación para instalar el conector de Brightcove en un sitio:

    1. Compruebe que la cuenta de usuario que está realizando este procedimiento es miembro del grupo Propietarios del sitio.
    2. La aplicación se puede agregar a un Sitio de una de dos maneras:
      1. En cualquier página, en el menú Configuración seleccione Agregar una opción de aplicación .
      2. Desde Contenido del sitio mediante la opción Agregar una aplicación .
    3. A continuación, haga clic en «Desde su organización» y, a continuación, haga clic en la aplicación BrightCoveConnector
    4. En el cuadro de diálogo Conceder permiso, haga clic en «Confiar en él».
    5. En este punto, se agrega la aplicación BrightCoveConnector para SharePoint y aparece en la sección Aplicaciones de la lista Contenido del sitio. Ahora está disponible para su uso en el Sitio.

    Aprovisionamiento de las listas predeterminadas para la aplicación Brightcove Video Connect

    • Cuando se crea una instancia de la aplicación por primera vez, o si las listas no están presentes en el Sitio local, el usuario recibirá hasta 3 opciones para permitir el aprovisionamiento de listas dentro del Sitio.

    • Cuando se hace clic, cada botón mostrará un mensaje al efecto:
    • Cuando se hace clic en «Crear lista de ajustes», además de crear la lista Configuración de Brightcove, se muestra un mensaje para editar esta lista para asegurarse de que la URL del proxy requerida para la compatibilidad con la API de Brightcove está configurada.
    • Al hacer clic en este enlace se abrirá la lista Configuración de Brightcove para editar inmediatamente la información de acceso al servidor proxy. El título debe establecerse en «Proxy». ProxyURL es específico de la ubicación del servidor proxy que utilizará para acceder a la API de Brightcove. Consulte Uso del servidor proxy para la comunicación de la API de Brightcove para obtener más detalles sobre el uso del servidor proxy.
    • Al completar el aprovisionamiento de listas se crearán las siguientes listas en el sitio:
      • Cuentas: lista personalizada que representa los objetos Account.
      • Vídeos: una lista personalizada que representa los objetos de vídeo.
      • Configuración - Lista para almacenar los detalles del servidor proxy.

    Instalación del paquete de soluciones de Reproductor para SharePoint Online

    El paquete de soluciones para Brightcove Player incluido en el paquete de Brightcove Video Connect permite al usuario incrustar reproductores de vídeo de Brightcove en las páginas del sitio de SharePoint. El paquete de soluciones de Brightcove Player utiliza controles de SharePoint, que son compatibles con SharePoint Online y SharePoint Server y, por lo tanto, se pueden utilizar con ambos. Una solución de entorno limitado de elementos Web Player para incrustar vídeo también está disponible en el paquete de Brightcove Video Connect, pero solo se puede utilizar con SharePoint Server en las instalaciones, ya que SharePoint Online y SharePoint Server alojado por Microsoft tienen restricciones de seguridad que prohíben las soluciones de entorno aislado; esta solución es descrito en la siguiente sección.

    Para incrustar un reproductor de Brightcove en SharePoint Online, el paquete de soluciones descrito aquí permite al usuario elegir un reproductor de vídeo o lista de reproducción y, a continuación, se genera código de incrustación del reproductor para que el usuario lo copie y pegue en el cuadro de diálogo Código de inserción de SharePoint.

    NOTA - Esta sección requiere el archivo de solución BrightCovePlayer.wsp.

    1. En el sitio raíz de la colección de sitios, donde está instalada la aplicación Brightcove, vaya a Configuración del > sitio - Administraciones de colecciones de sitios - Seguridad de campo > HTML.
    2. Agregar players.brightcove.net como dominio iframe permitido
    3. En el sitio raíz de la colección de sitios, donde está instalada la aplicación Brightcove, vaya a Configuración del sitio - Galerías del diseñador > web - > Soluciones.
    4. Cargue el archivo «BrightCovePlayer.wsp» en la Gallery de soluciones y active el archivo.
    5. Vaya a Configuración del sitio - Acciones > del sitio - > Administrar características del sitio.
    6. Active la función «Brightcove Player».
    7. Compruebe que la Biblioteca de documentos de BrightCoveConnectorResources se creó durante la instalación.
    8. De lo contrario, cree una nueva biblioteca de documentos llamada BrightCoveConnectorResources
    9. y cargue BrightcovePlayer.css y BrightcovePlayer.js a esa biblioteca de documentos.

    Instalación del servidor proxy para la comunicación de la API de Brightcove

    Dado que Connector es una aplicación JavaScript del lado del cliente, se requiere un servidor proxy para intermediar la comunicación autenticada con la API de Brightcove. Puede encontrar más información en el documento de la API de Brightcove.

    Se proporciona una aplicación de proxy .net en el paquete de Brightcove Video Connect. El servidor proxy debe implementarse en una ubicación accesible a la Web, como en Microsoft Azure. Por razones de seguridad, el proxy está diseñado para permitir únicamente conexiones desde sitios web incluidos en el archivo web.config instalado con el proxy. La clave brightcove:origins del archivo web.config debe incluir la dirección URL del sitio web de SharePoint para la aplicación BrightCoveConnector , así como la URL del sitio en el que se ejecutará el componente BrightCovePlayer . Vea el siguiente ejemplo:

    <add key="brightcove:origins" value="https://brightcovecms-a3da847a1c9e33.sharepoint.com, https://brightcovecms.sharepoint.com" />

    Varios sitios se pueden enumerar como una cadena separada por comas.

    Inicio de Brightcove Video Connect para SharePoint

    Para iniciar la aplicación, el usuario puede hacer clic en la aplicación BrightCoveConnector dentro de la lista Contenido del sitio, o hacer clic en un vínculo de menú si así lo configura el administrador de SharePoint.

    En cualquier caso, el usuario será llevado a la página principal de la aplicación Connector. Esta página podría ser configurada por el cliente para incluir mensajes adicionales específicos de la compañía, enlaces adicionales, etc. Tal como se describe en la sección de instalación, dependiendo de la pertenencia al grupo del usuario, es posible que vean o no las pestañas Administración de cuentas y Agregar vídeos.

    La navegación principal en la columna izquierda del Conector siempre está presente y permite al usuario navegar por todas las secciones a las que tiene acceso (ver la sección anterior sobre permisos). A continuación se muestra una breve descripción del propósito de cada componente del Conector:

    • Inicio : la página de destino cuando ingresa por primera vez al Conector.
    • Administración de cuentas : proporciona la interfaz para introducir la información de las cuentas de Brightcove Video Cloud en SharePoint.
    • Añadir vídeos : cargue vídeos nuevos en Video Cloud. También se crea un objeto de vídeo en SharePoint (excepto el archivo de vídeo real).
    • Administrar vídeos - Examine y edite vídeos existentes, así como importar vídeos de Video Cloud a SharePoint.
    • Administrar listas de reproducción - Crear nuevas listas de reproducción, así como navegar y administrar listas de reproducción existentes.

    También hay componentes Agregar vídeo desde Brightcove y Editar vídeo que se mostrarán en la navegación principal al realizar esas operaciones.

    Vinculación a cuentas de Video Cloud

    La sección Administración de cuentas de Connector se utiliza para introducir la información de cada una de las cuentas de Brightcove Video Cloud vinculadas a las que se tendrá acceso en el Sitio. Las cuentas de Brightcove no se pueden crear desde SharePoint, primero deben crearse en Video Cloud. La información de la cuenta de Brightcove debe introducirse en SharePoint tal como se utiliza al acceder a la API de Brightcove para comunicarse con Video Cloud. También proporciona los medios para asignar permisos de acceso específicos a una cuenta. Para clientes empresariales con varias cuentas, proporciona control sobre qué usuarios pueden acceder a cada cuenta y si solo pueden navegar o cargar vídeos.

    Página de inicio de administración de cuentas

    La página de inicio de administración de cuentas muestra la lista actual de cuentas configuradas, junto con los grupos asignados para Autores y Visores.

    1. Título de cuenta : cuentas configuradas actualmente (o al menos creadas).
    2. AuthorsGroup: grupo de SharePoint asignado derechos de «Autor» para la cuenta.
    3. ViewersGroup: grupo de SharePoint asignado derechos de «Visor» para la cuenta.
    4. Agregar nueva cuenta - Abra la pantalla de creación de cuenta para configurar una nueva cuenta.

    Crear/editar una cuenta

    Tanto si se crea una cuenta nueva como la edición de una cuenta existente, el proceso y las opciones son las mismas. Todas las propiedades de la cuenta (excepto los grupos Autor y Visores) deben obtenerse primero de la cuenta de Video Cloud. Las siguientes son las propiedades necesarias para cada cuenta.

    1. Nombre de cuenta : nombre de la cuenta. Este nombre solo se utiliza en SharePoint, por lo que no es necesario que sea el mismo que en Video Cloud.
    2. DefaultVideoPlayerID - Establece el reproductor predeterminado para los vídeos.
    3. DefaultPlayListPlayerID - Establece el reproductor de lista de reproducción predeterminado.
    4. ID de cuenta : el ID de cuenta de Brightcove de Video Cloud.
    5. ClientID : ID de cliente de autenticación de API de Brightcove.
    6. Secreto delcliente: el secreto del cliente de autenticación de API de Brightcove.
    7. Ubicación de almacenamiento temporal : ubicación de almacenamiento temporal que se utilizará durante la carga de vídeos y pistas de texto a Video Cloud. El menú desplegable permite seleccionar Amazon Web Services (AWS) o Dropbox.
    8. ID de clave de acceso de AWS : si utiliza AWS, el ID de clave de acceso se introduce aquí.
    9. Clave de acceso secreta de AWS : si utiliza AWS, la clave de acceso secreta se introduce aquí.
    10. Nombre del depósito de AWS : si utiliza AWS, el nombre del depósito de AWS se introduce aquí.
    11. ID de cuenta de Dropbox : si usas Dropbox, el ID de cuenta de Dropbox se introduce aquí.
    12. Token de acceso de Dropbox : si usas Dropbox, el token de acceso de Dropbox se introduce aquí.
    13. Campos personalizados : se utilizan para asignar los campos personalizados creados en Video Cloud para esta cuenta. Consulte la sección Asignación de campos personalizados para obtener más información.
    14. Grupo del autor, Grupo del visor : grupos de definición de permisos del lado de SharePoint para la cuenta.
      1. Los miembros del Grupo de Autores (en este caso el Grupo «Propietarios» de SharePoint) tienen la capacidad de realizar funciones en la cuenta que requieren un token de escritura, como agregar un video, editar y actualizar un vídeo existente, crear/editar una lista de reproducción, etc. También pueden realizar todas las funciones de tipo de lectura, como navegar y buscar vídeos y listas de reproducción.
      2. Los miembros del Grupo del Visor (en este caso, el Grupo «Visitantes» de SharePoint) solo están autorizados para funciones de tipo de lectura, como navegar y buscar vídeos y listas de reproducción.

    Para editar cualquiera de las propiedades de la lista de cuentas anterior, el usuario puede hacer clic en el nombre de la propiedad o en su valor (si está presente) para abrir el cuadro de diálogo emergente de la propiedad:

    Vinculación a Dropbox o AWS S3 para almacenamiento de carga temporal

    La API de Brightcove requiere que todos los archivos (archivos de vídeo, archivos de pistas de texto) que procese Video Cloud se carguen en una ubicación intermedia accesible tanto por el conector de Brightcove como por Video Cloud. AWS S3 y Dropbox son dos servicios recomendados que están disponibles en el conector de Brightcove.

    Puede encontrar instrucciones sobre la configuración de estos servicios para generar la información requerida en la sección Configuración de la cuenta del conector de Brightcove mediante los siguientes enlaces:

    • Visión general: API de ingesta dinámica
      Uso de Dynamic Ingest con S3
      http://docs.aws.amazon.com/AmazonS3/latest/gsg/CreatingABucket.html
      https://www.dropbox.com/developers

    NOTA: Los archivos cargados en AWS S3 o Dropbox no se eliminan automáticamente. Se recomienda configurar directivas en estos servicios para eliminar automáticamente los archivos después de un período de tiempo.

    Ventana emergente Ubicación de almacenamiento temporal

    Adición de pistas de texto

    Las pistas de texto se pueden agregar a cualquier vídeo en la primera carga o en actualizaciones posteriores. Un vídeo puede tener una o más pistas de texto asociadas a él. El usuario puede seleccionar el tipo de pista de texto que se agregará al vídeo (Carga de archivos, URL Subir o URL remota). Si se selecciona «Ninguno», la sección no mostrará ninguno de los campos.

    IMPORTANTE: Un solo vídeo NO admite una mezcla de subidas de archivos, subidas de URL o URL remotas. Sólo se puede asociar un tipo de pista de texto a cada vídeo.

    Subir desde archivo

    Cuando se selecciona «Carga de archivos» para Archivos de pista de texto, se muestran los siguientes campos:

    1. Seleccione Tipo de pista de texto: permite la selección de Carga de archivos, Cargar URL o URL remota.
    2. Selector de archivos - Selección del archivo de pista de texto.
    3. Propiedades de pista de texto: establece las propiedades asociadas a la pista de texto. Al hacer clic en «Añadir» se añade la pista de texto a los datos de vídeo en la próxima subida o actualización.
      1. Idioma - El código de dos letras (etiqueta de idioma BCP 47 válida) para el idioma de la pista de texto, por ejemplo «en» para inglés.
      2. Etiqueta: la etiqueta de la pista que será visible para el usuario, como en un menú que enumera los diferentes idiomas disponibles para los subtítulos.
      3. Tipo : uno de los cinco tipos de pistas admitidos enumerados
    4. Lista de pistas de texto añadidas/en cola: muestra la lista de pistas de texto existentes y nuevas pistas de texto que se colocan en cola para cargarlas en el siguiente guardado. Una pista de texto se puede eliminar haciendo clic en el enlace «Eliminar» y Guardar el vídeo.

    1. Seleccione Tipo de pista de texto: permite la selección de Carga de archivos, Cargar URL o URL remota.
    2. Agregar URL - Campo para introducir la ruta URL completa al archivo de pista de texto.
    3. Propiedades de pista de texto: permite la configuración de las propiedades asociadas a la pista de texto. Al hacer clic en «Añadir» se añade la pista de texto a los datos de vídeo en la próxima subida o actualización.
      1. Idioma - El código de dos letras (etiqueta de idioma BCP 47 válida) para el idioma de la pista de texto, por ejemplo «en» para inglés.
      2. Etiqueta - Etiqueta de la pista que se mostrará al usuario, como en un menú que enumera los diferentes idiomas disponibles para los subtítulos.

    4. Lista de pistas de texto añadidas/en cola: muestra la lista de pistas de texto existentes y nuevas pistas de texto que se colocan en cola para cargarlas en el siguiente guardado. Una pista de texto se puede eliminar haciendo clic en el enlace «Eliminar» y Guardar el vídeo.

    Administrar vídeos

    La sección Gestionar vídeos del Conector se utiliza para examinar y administrar vídeos existentes. Esto incluye la edición y actualización de vídeos existentes según sea necesario. La funcionalidad de búsqueda se utiliza para buscar vídeos por nombre de búsqueda, descripción e información de etiqueta.

    Página de destino Gestionar vídeos

    La página de inicio Gestionar vídeos muestra la lista de campos que se pueden utilizar para explorar o buscar vídeos en las cuentas disponibles:

    1. Seleccionar cuenta: esta lista desplegable se rellenará con todas las cuentas a las que el usuario esté autorizado a acceder; esto se habilita mediante la asignación Grupos de cuentas. El usuario debe seleccionar una cuenta para ver cualquier listado de vídeos.
    2. Buscar - Filtra los resultados del vídeo a los que contienen el término. Se busca información de nombre, etiquetas y descripción junto con los campos de metadatos de SharePoint, si se definen. Solo es posible buscar en una cuenta a la vez; no se admite la búsqueda cruzada entre cuentas.
    3. Mostrar sólo SharePoint o sólo Video Cloud : filtre los vídeos sólo a aquellos que están grabados en SharePoint, o aquellos que solo residan en Video Cloud y que nunca han tenido sus datos importados en SharePoint.
    4. Clasificar y ordenar filtros: se utilizan para ordenar y ordenar los resultados según los parámetros enumerados para ayudar a encontrar vídeos más rápidamente.

    1. Seleccionar cuenta : esta lista desplegable se rellenará con todas las cuentas a las que el usuario esté autorizado a acceder; t su está habilitada a través de la asignación Grupos de cuentas. El usuario debe seleccionar una cuenta para ver cualquier listado de vídeos.

    3. Clasificar y ordenar filtros: se utilizan para ordenar y ordenar los resultados según los parámetros enumerados para ayudar a encontrar vídeos más rápidamente.
    4. Controles de paginación : cada página mostrará hasta 5 listados de vídeos. Cuando hay más de 5 presentes, aparecerán los controles de paginación.
    5. Listado de vídeos - Cada listado de vídeo contiene lo siguiente:
      1. Nombre del vídeo - Al hacer clic en el nombre se abrirá el video para editar sus datos.
      2. Descripción abreviada
      3. Lista de las etiquetas asociadas al vídeo (no muestra el ID de referencia aunque esté guardado en Video Cloud como etiqueta).
      4. El enlace «Editar vídeo». Este enlace (o haciendo clic en el Nombre) se utiliza para abrir el vídeo para editarlo si el usuario actual tiene permiso.
      5. Id. de vídeo: Éste es el ID de vídeo de Video Cloud. Este ID aparecerá una vez editado el vídeo (incluso sin cambios) y guardado por primera vez después de la carga.

Para ello, primero busque el vídeo usando la opción de listado «Solo para Video Cloud» (1):

    La eliminación de vídeos de SharePoint se realiza fuera del conector. Para eliminar un vídeo en SharePoint, vaya a Contenido del sitio - > Vídeos, donde puede encontrar y eliminar el vídeo.

    IMPORTANTE: la eliminación del vídeo grabado en SharePoint no elimina el vídeo en Video Cloud y deja el vídeo en Video Cloud con algunas etiquetas que se usaron para vincular el vídeo a SharePoint.     2. Descripción - Una breve descripción de la lista de reproducción.

    4. Vídeos disponibles - En el caso de una lista de reproducción manual, muestre el cuadro de búsqueda y la lista de vídeos disponibles en la cuenta. Esto permite al usuario restringir las opciones de vídeo para agregar a la lista de reproducción.
    5. Listado de vídeos disponibles - Lista los vídeos disponibles en la cuenta o filtra según el término de búsqueda. Uno o más vídeos se pueden seleccionar haciendo clic en los vídeos. Los vídeos seleccionados se resaltan en verde. Para anular la selección de un vídeo, el usuario vuelve a hacer clic en él. Esto permite mover varios vídeos a la lista de reproducción en una sola acción.
    6. Flechas de acción : mueve los vídeos seleccionados dentro y fuera de la lista de reproducción.
    7. Vídeos de esta lista de reproducción: los vídeos actuales de la lista de reproducción se muestran aquí.
    8. Guardar : confirme los cambios e inicie el proceso de creación de la nueva lista de reproducción en Video Cloud. NOTA - Las listas de reproducción no se guardan en SharePoint.

    Edición de una lista de reproducción existente

    La edición de una lista de reproducción existente se inicia haciendo clic en el nombre o en el enlace «Editar» de la lista de reproducción. Esto abre la siguiente vista para editar las opciones de la lista de reproducción. En este ejemplo se muestran las opciones de una lista de reproducción inteligente. Al editar cualquiera de los campos, con la excepción del ID de la lista de reproducción y la cuenta, se pueden modificar:

    1. Nombre : el nombre de la lista de reproducción.
    2. Descripción - Una breve descripción de la lista de reproducción.
    3. Tipo - El tipo de lista de reproducción - Manual o Inteligente. En este ejemplo se tratan las opciones de una lista de reproducción inteligente.
    4. Etiquetas - En el caso de una lista de reproducción inteligente, este campo permite al usuario especificar criterios y las etiquetas a utilizar para identificar los vídeos que se van a incluir dinámicamente en la lista de reproducción.
      • El menú desplegable de criterios incluye «Contener uno o más» o «Contener todo».
    5. Orden - Esto permite al usuario especificar cómo se establece el orden de los vídeos coincidentes. Las selecciones incluyen lo siguiente:
      • Alfabético
      • Fecha de activación (la más reciente primero)
      • Fecha de activación (la más antigua primero)
      • Número total de reproducciones
      • Reproducciones más recientes de la semana
      • Fecha de comienzo (la más antigua primero)
      • Fecha de comienzo (la más reciente primero)
    6. Limitar a - Establece el número máximo de vídeos que se incluirán en la lista de reproducción en función de los criterios seleccionados.
    7. Vista previa - Enumera vídeos que coincidan con los criterios y los muestra en la ventana «Vídeos en esta lista de reproducción».
    8. Guardar : confirme los cambios e inicie el proceso de actualización de la lista de reproducción en Video Cloud. NOTA - Las listas de reproducción no se guardan en SharePoint.
    9. Eliminar lista de reproducción : esta opción marcará la lista de reproducción como eliminada en Video Cloud. La lista de reproducción ya no aparecerá en la lista de listas de reproducción del conector.

    Agregar vídeo a un sitio de SharePoint

    El conector de SharePoint permite a los autores de contenido insertar medios en sus sitios de diversas maneras. Este capítulo cubre las opciones disponibles.

    Elemento Web Brightcove Player para SharePoint en las instalaciones

    La solución de entorno limitado de elementos web de Brightcove Player genera un elemento Web que se coloca en una página de sitio en el sitio web interno o externo y reproduce el vídeo de Brightcove. SharePoint Online restringe la capacidad de los usuarios de agregar elementos Web personalizados, de modo que este mecanismo para insertar vídeos sólo está disponible para instalaciones de SharePoint locales.

    El elemento Web también se ha ampliado con algunas de las mismas características en la consola de administración, incluida la selección de cuentas y la búsqueda avanzada con clasificación y ordenamiento para encontrar vídeos y listas de reproducción más rápido.

    Para ver las opciones del elemento Web del Reproductor, la página de SharePoint debe estar en modo «Editar» (1), lo que expondrá la opción de editar el elemento Web (2). Las opciones específicas del elemento Web se encuentran en la sección «Configuración de Brightcove» (3) del editor de elementos Web.

    Al expandir la sección Configuración de Brightcove se muestran las características actualizadas de la solución de entorno aislado de elementos Web Player.

    Selección de un vídeo para reproducir en el elemento Web

    1. Configuración de Brightcove : esta sección del editor de elementos Web proporciona acceso a los parámetros para configurar el elemento Web y seleccionar los vídeos/listas de reproducción que se van a reproducir.
    2. Ancho del jugador, Altura del jugador - Establece las dimensiones del jugador.
    3. ID del jugador : especifique el ID del reproductor que se va a utilizar. Es importante que el reproductor seleccionado sea de la misma cuenta que el vídeo seleccionado si se trata de un entorno de varias cuentas. No hay ningún «selector» para el reproductor, por lo que el ID de Video Cloud debe introducirse manualmente.
    4. ID de vídeo : el campo ID de vídeo se rellena con un selector de vídeo. Si el usuario hace clic en el cuadro de elipses situado a la derecha del campo, se mostrará la ventana Selector.
    5. Ventana Selector
      • Seleccionar cuenta: muestra las cuentas a las que el usuario actual tiene permiso para acceder.
      • Vídeos disponibles - Filtra los resultados del vídeo a los que contienen el término. Se busca información de nombre, etiquetas y descripción junto con los campos de metadatos de SharePoint, si se definen. Solo es posible buscar en una cuenta a la vez; no se admite la búsqueda cruzada entre cuentas.
      • Ordenar por y ordenar por
      • Ventana de listado de vídeos - Al hacer clic en un vídeo se seleccionará para reproducirlo. El listado se limita a mostrar 30 vídeos, por lo que no todos los vídeos de la cuenta pueden aparecer al seleccionar una cuenta por primera vez. El uso de la funcionalidad de búsqueda permite refinar la lista para encontrar el vídeo deseado.
      • Botón Aceptar : confirma la selección y rellena el campo ID de vídeo con el valor de ID adecuado.

    Una vez que el ID del reproductor y el ID de vídeo están configurados, el elemento Web está configurado y listo para su reproducción.

    Selección de una lista de reproducción para reproducir en el elemento Web

    El proceso para seleccionar una lista de reproducción para reproducirse es casi idéntico al de seleccionar un vídeo.

    1. Configuración de Brightcove : esta sección del editor de elementos Web proporciona acceso a los parámetros para configurar el elemento Web y seleccionar los vídeos/listas de reproducción que se van a reproducir.
    2. Ancho del jugador, Altura del jugador - Establece las dimensiones del jugador.
    3. ID del jugador : especifique el ID del reproductor de lista de reproducción que se va a utilizar. Es importante que el reproductor seleccionado sea de la misma cuenta que el vídeo seleccionado si se trata de un entorno de varias cuentas y que esté configurado para mostrar una lista de reproducción. No hay ningún «selector» para el reproductor, por lo que el ID de Video Cloud debe introducirse manualmente.
    4. ID de lista de reproducción : el ID de vídeo se rellena con un selector de vídeo. Si el usuario hace clic en el cuadro de elipses situado a la derecha del campo, se mostrará la ventana Selector.
    5. Ventana Selector
      • Seleccionar cuenta: muestra las cuentas a las que el usuario actual tiene permiso para acceder.
      • Ordenar por y ordenar por
      • Ventana de lista de reproducción - Al hacer clic en una lista de reproducción se seleccionará para jugar. El listado está limitado a mostrar 30 listas de reproducción, por lo que no todas las listas de reproducción de la cuenta pueden aparecer al seleccionar una cuenta por primera vez. El uso de la funcionalidad de búsqueda permite refinar la lista para encontrar la lista de reproducción deseada.
      • Botón Aceptar : confirma la selección y rellena el campo ID de la lista de reproducción con el valor de ID apropiado.

    Una vez que el ID de la lista de reproducción y el ID de vídeo están configurados, el elemento Web está configurado y listo para reproducirse.
