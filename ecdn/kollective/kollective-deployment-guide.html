    Guía de implementación de eCDN de Kollective

    En este tema, aprenderá cómo integrar el eCDN Kollective con Brightcove Player.

    Resumen

    Brightcove ha creado una integración para entregar eventos Video On Demand (VOD) y Brightcove Live a los reproductores de Brightcove mediante el eCDN Kollective.

    Una solución eCDN ayuda con la entrega de contenido a las redes corporativas. Alivian los cuellos de botella de la red cuando tiene un gran número de visores simultáneos que ven el mismo contenido, como en las oficinas corporativas. Con eCDN, puede descargar contenido de vídeo del servidor de origen una vez para cada instancia del servidor eCDN. Los visores de la red corporativa reciben el contenido del servidor eCDN a través de la LAN.

    Algunas de las ventajas de utilizar un eCDN incluyen las siguientes:

    • Reducción del tráfico WAN para ubicaciones corporativas
    • Experiencia de visualización mejorada a escala
    • Reducción de los costos de red
    • Mayor seguridad ya que el eCDN está detrás del cortafuegos corporativo
    • Supervisión del rendimiento

    Requisitos

    Los siguientes requisitos se aplican al complemento eCDN:

    • Debe utilizar una versión 6.42.2+ de Brightcove Player
    • Los vídeos deben ser ingeridos para la entrega dinámica

    Introducción

    Para usar el plugin kollective, necesitará alguna configuración inicial:

    1. Implementación de la aplicación del lado del servidor
    2. Instalación del plugin del reproductor
    3. Incluir nombres de dominio en listas blancas

    4. Ahora que ha completado los pasos de implementación, está listo para sincronizar el contenido e incrustar el código del reproductor. Para obtener más información, consulte lo siguiente:

      • Guía del usuario de Kollective eCDN

    Implementación de la aplicación del lado del servidor

    La aplicación ECDN backend debe implementarse en el servidor.

    Requisitos del servidor

    Los siguientes requisitos se aplican al servidor de alojamiento para la aplicación back-end.

    • NodeJS versión 13+
    • Servidor de alojamiento: Servidor Linux o Windows (la aplicación backend también se puede implementar en Heroku)
    • Base de datos MongoDB

    El MongoDB puede estar en el mismo servidor que el servidor de alojamiento, en otro servidor o usando una base de datos en la nube MongoDB Atlas.

    Configuración de configuraciones

    Realizará algunos cambios de configuración para la aplicación back-end. Siga estos pasos:

    1. En el directorio raíz del paquete de implementación de aplicaciones back-end, abra el archivo config.env .

    2. Modo de producción - Establezca el modo para ejecutar la aplicación en producción o desarrollo.

      Tampoco 

      NODE_ENV=production

      o bien 

      NODE_ENV=development

    3. Puerto de aplicación - Establezca el puerto en el que se ejecutará la aplicación.

      Ejemplo: 

      PORT=3000

    4. Base de datos - La aplicación back-end requiere una base de datos MongoDB.

      Puede especificar la conexión de base de datos con el siguiente formato: 

      DATABASE=mongodb://{host}:{port}/{databaseName}

      Ejemplo: 

      DATABASE=mongodb://localhost:27017/Brightcove

      Si desea utilizar una base de datos en la nube MongoDB Atlas, puede configurarla de la siguiente manera: 

      DATABASE=mongodb+srv://{user}:{password}@{cluster}/{databaseName}?retryWrites=true&w=majority

      Ejemplo: 

      DATABASE=mongodb+srv://myuser:mypassword@cluster0-xafsz.mongodb.net/MyDatabase?retryWrites=true&w=majority

    5. Servicio de correo electrónico : la aplicación back-end requiere una configuración de servicio de correo electrónico para enviar correos electrónicos de notificación.

      SendGrid

      Para enviar correos electrónicos mediante el servicio de correo electrónico SendGrid, utilice lo siguiente junto con su nombre de usuario y contraseña de SendGrid: 

      # Send email using Sendgrid email service.
EMAIL_SERVICE=SendGrid

# Send email using Sendgrid email service.
SENDGRID_USERNAME=your SendGrid username
SENDGRID_PASSWORD=your SendGrid password
      Otro

      Para enviar correos electrónicos mediante otros servicios de correo electrónico, utilice lo siguiente con el host del servidor de correo electrónico, puerto y nombre de usuario, contraseña: 

      # Set EMAIL_SERVICE=Other to send email using normal email service using host and port
EMAIL_SERVICE=Other

# Send email using host and port
EMAIL_HOST=your email host
EMAIL_PORT=your email port
EMAIL_USERNAME=your email username
EMAIL_PASSWORD=your email password
EMAIL_FROM=your email from address

    6. URL base de la API de publicación de Kollective : la aplicación del lado del servidor llama a la API de publicación de Kollective para publicar un vídeo desde Brightcove con fuentes de vídeo en Kollective y obtener un token de contenido. Aquí, especifique la URL base de la API de publicación de Kollective: 

      # Kollective publish API base url
KOLLECTIVE_PUBLISH_API_BASE_URL=https://content.stage.kollective.app/api

    7. URL base de la API de Brightcove : la aplicación del lado del servidor necesita las siguientes direcciones URL base de la API de Brightcove: 

      # Brightcove API base URLs
BRIGHTCOVE_ACCESS_TOKEN_BASE_URL=https://oauth.brightcove.com/v4/access_token
BRIGHTCOVE_CMS_API_BASE_URL=https://cms.api.brightcove.com/v1
BRIGHTCOVE_LIVE_API_BASE_URL=https://api.bcovlive.io/v1

    8. Nombres de campo personalizados : no se pueden cambiar los siguientes parámetros de nombre de campos personalizados: 

      # Required custom field names for each Brightcove Account ID
CUSTOM_FIELD_KOLLECTIVE_CONTENT_TOKEN=kollective_content_token
CUSTOM_FIELD_KOLLECTIVE_IS_SYNCHRONIZED=kollective_is_synchronized
CUSTOM_FIELD_KOLLECTIVE_SYNCHRONIZED_TIME=kollective_synchronized_time
CUSTOM_FIELD_KOLLECTIVE_LAST_SYNC_STATUS=kollective_last_sync_status

    Agregar campos personalizados

    A continuación, está listo para definir los campos personalizados necesarios en Video Cloud Studio.

    1. En Video Cloud Studio, expanda el menú desplegable Administrador y seleccione Campos de vídeo.

    2. En la página Campos de vídeo, seleccione el botón Agregar campo personalizado .

    3. Aquí, agregará campos personalizados requeridos para Kollective.

      Cada ID de cuenta de Brightcove que se va a utilizar para una cuenta de sindicador necesita los siguientes nombres de campo personalizados definidos para que funcione la sincronización:

      Resumen de la función de balizas de Brightcove
      Nombre del campo personalizado Tipo
      kollective_content_token text
      kollective_is_synchronized text
      kollective_synized_time text
      kollective_last_sync_status text

      En la página Detalles del campo de vídeo, introduzca lo siguiente:

      • Nombre para mostrar: introduzca los nombres de campo personalizados de la tabla anterior. Guarde cada campo por separado.
      • Nombre interno: se genera a partir del nombre para mostrar.
      • Deje el tipo como texto
      • Introducir una descripción
      • Deje la opción Hacer que este campo sea obligatorio NO seleccionada

    Implementación de la aplicación

    Después de realizar todos los cambios de configuración necesarios y agregar los campos personalizados requeridos, estará listo para implementar la aplicación back-end en el servidor.

    1. Copie el paquete de aplicación del lado del servidor en la carpeta de implementación de destino del servidor de alojamiento.

    2. Abra una aplicación de consola y vaya a la carpeta raíz del paquete de aplicación en el paso anterior. Ejecute el siguiente comando para instalar todos los paquetes de Node necesarios: 

      # npm install --production

    3. Ejecute el siguiente comando para importar el usuario super-admin predeterminado: 

      # npm run import-users

    4. Para iniciar la aplicación backend, ejecute el siguiente comando: 

      # npm run start

    5. Si todo está configurado correctamente, puede abrir la aplicación en un navegador usando la dirección configurada.

      Ejemplo: 

      http://localhost:3000/login

    6. Inicie sesión en la aplicación con el usuario super-admin predeterminado importado anteriormente.

      • Nombre de usuario: super-admin@brightcove.com
      • Contraseña: BR1ghtC0ve

    Instalación del plugin del reproductor

    Para integrar Kollective con Brightcove Player, agregue el plugin Kollective a su reproductor.

    El complemento Kollective es compatible con la versión v6.45+ de Brightcove Player.

    Los siguientes pasos detallan el uso de Video Cloud Studio para cargar el complemento.

    1. En Video Cloud Studio, abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea agregar el plugin.
    2. Seleccione el enlace para que el reproductor abra las propiedades del reproductor.
    3. Seleccione Plugins en el menú de navegación de la izquierda.

    4. A continuación, seleccione el botón Agregar un complemento y, a continuación, seleccione Brightcove Plugin.

    5. Expanda el menú desplegable Brightcove Plugin y seleccione kollective.

    6. Para las opciones (JSON): Introduzca lo siguiente:

      • ksdkUrl - Puntos a la ksdk biblioteca de Kollective — utilice la versión de su elección 
      {
    "ksdkUrl": "ksdk URL”
}
    7. Seleccione el botón Guardar.
    8. Publica tu reproductor.
    9. Actualizar/cambiar el código de inserción de data-video-id a data-kollective-video-id.
      10. Al incrustar un reproductor para su uso con la integración de Kollective, debe actualizar/cambiar el código de inserción de data-video-id a data-kollective-video-id.

    Incluir nombres de dominio en listas blancas

    Kollective necesita agregar sus nombres de dominio a su lista blanca para que no aparezca un error CORS. Proporcione los siguientes nombres de dominio a Kollective:

    • El nombre de dominio en el que implementó la aplicación del lado del servidor

    • El dominio en el que implementó sus páginas web

    Por ejemplo, si tiene una página en https://my-server.com/my-video-page que utiliza Brightcove Player con el complemento Kollective, entonces el dominio my-server.com debe agregarse a la lista blanca de Kollective. Esto permitirá la reproducción en la página sin devolver un error CORS.
