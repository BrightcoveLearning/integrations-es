Conector Adobe-AEM-Brightcove: Configuración

Este tema cubre la configuración del conector Brightcove para Adobe Experience Manager (AEM).

Introducción

El conector de Adobe-AEM-Brightcove le permite administrar videos y reproductores de Brightcove Video Cloud dentro de AEM, e incrustar videos fácilmente en páginas de AEM.

Notas:
  • Si está utilizando Adobe Experience Cloud, asegúrese de ver el Adobe Experience Cloud sección para requisitos de configuración adicionales /
  • En esta guía se utiliza AEM 6.3. El conector también funciona con AEM 6.2, 6.4 y 6.5 y los pasos son similares.

Pasos de configuración

Las siguientes secciones lo guiarán a través de los pasos para configurar el conector Adobe-AEM-Brightcove mediante la interfaz clásica. Tenga en cuenta que antes de completar estos pasos, el conector debe estar instalado; consulte Conector Adobe-AEM-Brightcove: Instalación

Deshabilitar la transcodificación DAM

De forma predeterminada, cuando carga activos de video en AEM DAM, se ejecutará ffmpeg en segundo plano para transcodificar el video en una variedad de formatos.  Si no necesita los activos transcodificados en el DAM, debe desactivar la transcodificación para ahorrar tiempo de procesamiento y almacenamiento:

  1. Abra las herramientas de AEM:
  2. Seleccionar flujo de trabajo:
  3. Seleccionar modelos:
  4. Seleccionar vista de lista:
  5. Seleccione Activo de actualización de DAM:
  6. Seleccione Editar:
  7. Haga doble clic en Transcodificación FFmpeg desde el flujo de trabajo para editarlo:
  8. Seleccione la pestaña Proceso en el cuadro de diálogo Propiedades del paso:
  9. Cambiar el proceso a No operacion:
  10. Cambiar Ok para cerrar el cuadro de diálogo y luego guardar los cambios

Si necesita los recursos de video transcodificados localmente, deberá asegurarse de que ffmpeg esté instalado en su servidor AEM:

  1. Verifica que tienes FFmpeg instalado abriendo una línea de comando y escribiendo: 
    ffmpeg
  2. Si el ffmpeg no se encuentra el comando, debe instalarlo:

Introduzca la información de la cuenta

Ahora está listo para configurar el conector AEM-Brightcove con la información de la cuenta de Video Cloud.

  1. Abra el administrador de configuración de la consola web de AEM (http: // localhost: 4502 / system / console / configMgr)
  2. Busca en Servicio Brightcove para localizar el servicio:
  3. Haga clic en el + firmar para agregar una nueva configuración:
  4. En el cuadro de diálogo, ingrese los valores como se muestra a continuación. Ver Administrar las credenciales de autenticación de API para obtener instrucciones sobre cómo obtener una identificación de cliente y un secreto de cliente. Los permisos que necesita aquí son:
    • A continuación se muestran los valores sugeridos para el formulario de configuración:
      • Alias de cuenta: introduzca un nombre para la cuenta que se mostrará en el conector (obligatorio)
      • ID de cuenta: introduce tu ID de editor de Brightcove (obligatorio)
      • ID de cliente: introduzca su ID de cliente de Brightcove desde la página Autenticación de la API de Brightcove (obligatorio)
      • Secreto de cliente: introduzca su API secreta de cliente de Brightcove (obligatorio)
      • Grupos permitidos: especifique el grupo que tendrá acceso al conector (obligatorio) asegúrese de que se trata de un grupo en el que está incluido.
      • Ruta de la tienda de jugadores: mantenga el valor predeterminado o utilice la carpeta de su elección para los jugadores de esta cuenta de Brightcove (obligatorio)
      • ID de reproductor de vídeo predeterminado: mantenga el valor predeterminado o elija otro ID de reproductor (obligatorio)
      • Clave predeterminada del reproductor de vídeo: obsoleta, deja en blanco o usa el valor del conector anterior
      • ID de jugador de lista de reproducción predeterminada: introduce un ID de jugador habilitado para las listas de reproducción si planeas usar listas de reproducción (obligatorio)
      • Clave de reproductor de lista de reproducción predeterminada: obsoleta, deja en blanco o usa el valor del conector anterior
      • Servidor proxy: deje en blanco o introduzca proxy si utilizaba anteriormente un proxy
      • Ruta de integración de Dam: mantenga el valor predeterminado o utilice la carpeta de su elección para los activos de vídeo de esta cuenta de Brightcove (obligatorio)
      • Perfil de ingesta: especifique el perfil de ingesta de Brightcove que se utilizará para los vídeos subidos, el valor en blanco utilizará el perfil predeterminado, si no funciona en blanco o por defecto, solicite al soporte de Brightcove que se asegure de que se defina un perfil predeterminado en el backend de Video Cloud
  5. Prueba tu configuración.  Vaya a http: // localhost: 4502 / brightcove / admin y seleccione la cuenta de Brightcove que acaba de agregar en el menú desplegable de la esquina superior derecha.  Si no se muestran videos, es probable que se deba a una de las siguientes razones:
    1. Tus credenciales de API ID de la cuenta , Identificación del cliente , y Secreto del cliente puede haber sido ingresado incorrectamente en la configuración del conector
    2. Sus credenciales de API no se configuraron correctamente en Brightcove
    3. El usuario que ha iniciado sesión actualmente no es miembro de uno de los Grupos permitidos en la configuración del conector
    4. Su configuración de red está bloqueando el acceso de la API a los servidores en la nube de Brightcove. Ver "Problemas de depuración de red y conexión de API" para más información.
  6. Si va a utilizar varias cuentas de Video Cloud, repita los pasos anteriores para cada cuenta.

Configurar Brightcove Players

La última tarea de configuración es configurar Brightcove Players.

  1. Navegue a las herramientas de administración del sitio (http: // localhost: 4502 / siteadmin # / content en la IU clásica o http: // localhost: 4502 / sites.html / content en la IU táctil)
  2. En la carpeta Sitios web de nivel superior, cree una nueva carpeta que coincida con el Ruta de la tienda de jugadores en la configuración.  El valor predeterminado es una carpeta. el nombre en la configuración se llama Brightcovetools con una carpeta dentro llamada jugadores.  Si se está conectando a varias cuentas de Brightcove, es posible que desee crear una carpeta de jugadores única para cada cuenta.
  3. Dentro de jugadores carpeta cree una nueva página de Brightcove Player:
  4. Después de crear la página, haga clic con el botón derecho en ella y seleccione Propiedades en el menú de la IU clásica; en la interfaz de usuario táctil, seleccione la página y luego haga clic en Propiedades.
  5. Vaya a la pestaña Brightcove Basic del cuadro de diálogo y a la cuenta Brightcove y luego seleccione un jugador de la lista:
  6. Si también desea que varios reproductores estén disponibles para su publicación, repita los pasos anteriores para crear una página para cada jugador.
  7. Si también desea utilizar listas de reproducción, repita los pasos anteriores para crear una página para un reproductor de listas de reproducción.
  8. Si está utilizando varias cuentas de Brightcove, deberá crear páginas de reproductor para cada una.

  1. Abra las herramientas de AEM:
  2. Seleccionar activos:
  3. Seleccione esquemas de metadatos:
  4. Seleccione Brightcove y haga clic en Aplicar a carpetas:

  6. Ingrese el nombre "brightcove_assets":
  7. Seleccione brightcove_assets y haga clic en Aplicar:

Adobe Experience Cloud

Hay algunos cambios de configuración necesarios para utilizar el conector Brightcove en AEM Cloud.

  1. Incluya como parte de su implementación una configuración OSGI para definir la información de su cuenta. Tiene la misma información que las versiones anteriores de Brightcove, pero debe implementarse en su proyecto ui.config o similar.
  2. Incluya como parte de su implementación una configuración OSGI para el oyente de replicación si desea sincronizar las actualizaciones de activos con Brightcove cuando las publique. Esto reemplaza al Agente de replicación personalizado en desuso incluido en versiones anteriores de Connector.
  3. Asegúrese de habilitar el iniciador de flujo de trabajo de fecha de última modificación de DAM para cada carpeta de cuenta de Brightcove en el DAM. Esto está deshabilitado de forma predeterminada en las últimas instancias de Cloud.
  4. Al igual que en las versiones anteriores, el conector viene con una pestaña de esquema de metadatos automático (opcional). En lugar de aplicar esto automáticamente, debe seleccionar este esquema y aplicarlo a las carpetas de Brightcove manualmente.
  5. Esta actualización requiere Java 11. Actualice su versión si es necesario.
  6. Deberá incrustar este paquete de dependencia en sus proyectos de Maven. Los paquetes deben ser referenciados como dependencias en pom.xml, según lo proporcione el proveedor.
  7. Para hacer esto, puede usar un repositorio de archivos local dentro de su pom.xml como el siguiente ejemplo: 
    <repositorios> <repositorio> <id>my-local-repo</id > <url>file: //${project.basedir}/my-repo</url > </repositorio > </repositorios >
  8. Luego, puede usar el siguiente comando para instalar el paquete del conector: 
    mvn org.apache.maven.plugins:maven-install-plugin:2.3.1: archivo de instalación\ -Darchivo=<ruta a archivo> -dGroupID=com.coreSecure\ -DartifactId=brightcove.connector -Dversion=1.0.0\ -dPackaging=JAR -DLocalRepositoryPath=<ruta definida anterio>
  9. Finalmente, debe hacer referencia al paquete instalado localmente a través de la sección de dependencias de pom.xml: 
    <dependencia > <GroupID > com.coreSecure</GroupID > <ArtifactID > Brightcove.connector</ArtifactID > <versión>1.0.0</versión > </dependencia >
